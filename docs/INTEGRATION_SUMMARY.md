## 🔗 Связь между компонентами

### Auth Flow
1. Пользователь вводит данные → `handleLogin()`/`handleRegister()`
2. frontend-api.js → `/api/auth/login` или `/api/auth/register`
3. Backend возвращает токен → сохраняется в localStorage
4. Обновляется UI пользователя

### Generation Flow
1. Пользователь заполняет wizard → `startGen()`
2. frontend-api.js → `/api/generate/async`
3. Backend создает проект → запускает FLUX.1
4. Frontend поллит статус → `pollGenerationStatus()`
5. Генерация завершена → `renderResults()`

### Projects Flow
1. Пользователь открывает dashboard → `renderProjects()`
2. frontend-api.js → `/api/projects`
3. Backend возвращает проекты → отображается в grid
