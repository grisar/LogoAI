# Инструкция по запуску LogoAI

## 🚀 Быстрый старт

### 1. Настройка Backend

```bash
cd backend

# Скопируйте шаблон .env
cp .env.example .env

# Отредактируйте .env и добавьте:
# - REPLICATE_API_TOKEN (получить на https://replicate.com/account/api-tokens)
# - JWT_SECRET (любой секретный ключ, например: super-secret-key-12345)
```

### 2. Инициализация базы данных

База данных уже инициализирована! ✅
- База данных: `logoai`
- Пользователь: `postgres`
- Пароль: `postgres`
- Порт: `5432`

### 3. Запуск Backend

```bash
cd backend
npm start
```

Backend будет доступен на `http://localhost:3000`

### 4. Запуск Frontend

Есть два варианта:

#### Вариант A: Использовать интегрированный HTML файл

```bash
cd /root/figma
# Скопируйте frontend-api.js в ту же директорию
cp backend/frontend-api.js .

# Откройте интегрированный файл в браузере
# Дважды кликните на logoai-prototype-integrated.html
```

#### Вариант B: Использовать httpd

```bash
# Если httpd установлен, запустите его
systemctl start httpd

# Скопируйте файлы в директорию веб-сервера
cp logoai-prototype-integrated.html /var/www/html/index.html
cp backend/frontend-api.js /var/www/html/

# Откройте в браузере
# http://localhost
```

## 📝 Регистрация и вход

1. Откройте HTML файл в браузере
2. Нажмите "Войти" или "Начать бесплатно"
3. Заполните форму регистрации:
   - Email: любой email (например, test@example.com)
   - Пароль: любой пароль (минимум 6 символов)
4. Нажмите "Зарегистрироваться"
5. Вы будете автоматически авторизованы

## 🎨 Создание логотипа

1. После входа нажмите "Новый логотип" или перейдите в wizard
2. Заполните форму:
   - Название бренда
   - Ниша (например: Технологии, Дизайн, Медицина)
   - Стиль (например: Минималистичный, Геометрический)
3. Нажмите "Сгенерировать"
4. Подождите 3-5 секунд (FLUX.1 генерирует логотипы)
5. Выберите понравившийся вариант
6. Редактируйте или сохраните в избранное

## 🔑 Получение REPLICATE API TOKEN

1. Перейдите на https://replicate.com
2. Создайте аккаунт (бесплатно)
3. Перейдите в https://replicate.com/account/api-tokens
4. Нажмите "Create API Token"
5. Скопируйте токен
6. Добавьте в `.env` файл:

```bash
REPLICATE_API_TOKEN=r8_ваш_токен_сюда
```

## 🐛 Возможные проблемы

### CORS ошибки

Если возникают CORS ошибки:
1. Убедитесь, что бэкенд запущен (`npm start`)
2. Проверьте, что `FRONTEND_URL` в `.env` соответствует вашему фронтенду
3. Попробуйте открыть HTML файл через httpd вместо прямого открытия файла

### Ошибка аутентификации (401)

Если возникает 401 ошибка:
1. Убедитесь, что вы вошли в аккаунт
2. Проверьте консоль браузера на наличие ошибок
3. Попробуйте выйти и войти снова

### Ошибка генерации (402 Payment Required)

Если генерация недоступна:
1. Проверьте свой тариф и количество генераций
2. Перейдите в "Подписка" для просмотра статистики
3. При необходимости обновите тариф

### Генерация не работает

Если логотипы не генерируются:
1. Убедитесь, что `REPLICATE_API_TOKEN` валиден в `.env`
2. Проверьте баланс аккаунта Replicate
3. Посмотрите логи бэкенда на наличие ошибок
4. Попробуйте повторить генерацию

## 🛠️ API Эндпоинты

### Аутентификация
- `POST /api/auth/register` - Регистрация
- `POST /api/auth/login` - Вход

### Пользователи
- `GET /api/users/me` - Профиль
- `PATCH /api/users/me` - Обновление профиля
- `DELETE /api/users/me` - Удаление аккаунта

### Проекты
- `GET /api/projects` - Список проектов
- `GET /api/projects/:id` - Детали проекта
- `POST /api/projects` - Создать проект
- `PATCH /api/projects/:id` - Обновить проект
- `DELETE /api/projects/:id` - Удалить проект
- `POST /api/projects/:id/favorite` - Избранное

### Генерация
- `POST /api/generate/sync` - Синхронная генерация
- `POST /api/generate/async` - Асинхронная генерация
- `GET /api/generate/status/:predictionId` - Статус генерации
- `DELETE /api/generate/cancel/:predictionId` - Отмена генерации

### Подписки
- `GET /api/subscriptions/me` - Текущая подписка
- `POST /api/subscriptions/checkout` - Создать оплату
- `POST /api/subscriptions/cancel` - Отменить подписку
- `GET /api/subscriptions/invoices` - История платежей

## 🔧 Настройка для Production

Для продакшена:

1. **Безопасность**
   - Замените `JWT_SECRET` на сильный ключ
   - Используйте HTTPS
   - Настройте Rate Limiting
   - Добавьте CSRF защиту

2. **Масштабирование**
   - Используйте PM2: `pm2 start backend/server.js`
   - Настройте Nginx как reverse proxy
   - Добавьте кэширование (Redis)

3. **Мониторинг**
   - Настройте логирование
   - Добавьте метрики (Prometheus)
   - Используйте APM (New Relic, Datadog)

4. **Платежи**
   - Интегрируйте Stripe
   - Настройте webhooks
   - Добавьте рекуррентные платежи
